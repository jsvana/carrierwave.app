2026-04-03 Update

iOS State of the Union

Hey folks! First, thanks everyone for giving our app a shot. Justin and I are so excited to finally be sharing it with people. We've loved making it and using it, and it's fulfilling to have folks be able to enjoy it with us.

We've shipped five releases recently. We've worked through a comprehensive manual rewrite, and we're finally live on the Apple App Store. Here's a look at everything that's landed since we went public.

What shipped

March and early April were some of the most productive weeks since development started. Five point releases landed:

1.76.0 — Radio Control & Session Management

  • WiFi radio control. Carrier Wave can now connect to your rig over WiFi in addition to Bluetooth LE. WiFi is more reliable during transmit (no BLE dropouts) and supports higher data throughput for scope and waterfall data. Icom CI-V radios like the IC-705 work in both Access Point and Station mode.
  • Session templates. Save a session configuration — mode, frequency, equipment, activation type — as a named template and recall it with a tap. Separate from “Save as Defaults” so you can keep multiple setups for different operating scenarios.
  • SMS spotting. For off-grid POTA and SOTA activations where you have cell service but no data, you can now self-spot via text message.
  • Repeating auto-spot. SPOT <message> REPEAT <min> keeps your spot fresh on the network. SPOT STOP cancels it.
  • Session QSO comments. Set a comment that gets prepended to every QSO's notes — useful for contest exchanges or consistent activation notes.
  • Edit session callsign. Change the station callsign on an existing session and it propagates to all QSOs in that session.
  • Experiments section in Settings. Alpha and beta features now get maturity badges so you know what to expect.

1.77.0 — Spot Filtering

  • Persistent filters. HUNT and P2P filter selections now save across sessions. Active filters show filled icons and a descriptive summary (e.g., “20m, CW, POTA only”).
  • Club and friends filtering. In HUNT mode, filter spots to show only members of your clubs or people on your friends list.
  • Activity filter. Narrow hunt spots by program — POTA, SOTA, or all.
  • P2P band/mode filters. Multi-select band and mode filters for park-to-park spots.
  • ADIF import into sessions. Long-press any session to import an ADIF file directly, merging contacts with validation warnings for callsign or park reference mismatches.

1.78.0 — Equipment Catalog & FlexRadio

  • Equipment Catalog. A structured equipment inventory replaces the old string-based lists. Browse a server-maintained catalog with detailed specs — bands, modes, portability, power — or add custom equipment. Fuzzy matching and “did you mean?” suggestions make finding your rig fast. Everything syncs via iCloud.
  • Activation Recommendations (Alpha). An engine that analyzes propagation, proximity, award progress, gear freshness, park novelty, and community interest to suggest your next activation. A serendipity slider lets you balance practical vs. adventurous. Milestone gravity prioritizes award advancement when you're close.
  • Log Scanning (Alpha). Point your camera at a handwritten paper log and import it. On-device Vision OCR recognizes text, feeds it through the quick entry parser, and lets you review before importing.
  • FlexRadio support. LAN transport for FlexRadio Systems transceivers (FLEX-6400, FLEX-6600, FLEX-6700) with automatic discovery via Vita-49 broadcast and SmartLink remote access. That brings supported connection transports to three: BLE, WiFi, and LAN.
  • WebSDR Instant Replay. Replay the last 15 seconds of captured audio with a button tap or Logger commands (REPLAY, AGN, HUH). Buffered playback doesn't interrupt the live connection.
  • QRZ multi-callsign support. Operators with multiple callsigns can now associate separate QRZ API keys with each one. QSOs are automatically routed to the correct logbook based on the myCallsign field.

1.79.0 — Sync Engine & Widgets

  • Per-service sync modes. Each connected service — QRZ, LoFi, Club Log, eQSL — can now be set to Bidirectional, Upload Only, or Download Only. You control exactly what data flows where.
  • Journal sync engine. A new iCloud sync option alongside CKSyncEngine. New devices download a single SQLite snapshot instead of syncing thousands of individual records, then catch up via batched journals. Ideal for large logs (10,000+ QSOs) where per-record sync is slow on initial setup. Automatic snapshots refresh every 6 hours.
  • Widget redesign. Radio Spots widgets now support SOTA alongside POTA and RBN. New activity type icons (tree, mountain, beacon), progressive detail by widget size, and signal strength info for RBN spots.

1.80.0 — Session Detail & IC-705 WiFi

  • Session detail pane rework. The session detail view has been redesigned for a cleaner layout and faster access to the information that matters during and after an operating session.
  • IC-705 WiFi rig control. Native WiFi CAT control for the Icom IC-705, letting you connect and control the radio over your local network without Bluetooth.

Beyond the app

The last few weeks weren't just code. A few things happened around the product:

  • App Store launch. Carrier Wave went live at $4.99. We updated the website, removed “Coming Soon” messaging, and pointed everything at the real App Store listing.
  • Comprehensive manual rewrite. The documentation got a ground-up overhaul — 9 new reference pages (FT8, Radio Control, Activations, Contests, Smart Needs, Brag Sheets, Video Export, Events, Onboarding), and 17 existing pages updated. Three persona-specific paths guide new users, beginners, and power operators through different depths of the app.
  • Challenges removed. We pulled the Challenges feature entirely. It was duplicating in-app statistics and wasn't adding enough value for the complexity it introduced, and we'd rather focus on the features operators actually use every day.
  • Website cleanup. Removed open-source claims and GitHub links from the site to better reflect the current direction of the project.
  • Product rename. Carrier Wave as you know it will become “Carrier Wave: Field”. There are a few reasons for doing this, but mainly because Field is not the only program we're working on. For you Mac users, we're currently working on a desktop suite named “Carrier Wave: Sweep” but more to come on that later.

Where we are

Carrier Wave: Field is at v1.80.0 with a feature set that covers the full lifecycle of an operating session: rig control, logging, spotting, syncing, activations, contests, social features, and analytics. The app talks to QRZ, POTA, LoTW, HAMRS, and eQSL. It runs on iPhone, iPad, and Apple Watch.

That's a lot of surface area — and it's the right time to stop adding to it for a while.

What's next: stabilization

Starting now, the focus shifts from new features to making everything that's already there rock solid. Concretely, that means:

  • Bug fixes over features. If it crashes, drops a connection, or loses data, that gets fixed first. New feature work takes a back seat.
  • Sync reliability. With seven services connected, sync edge cases are the biggest source of user-reported issues. We're going to methodically work through the backlog.
  • Performance. Large logs, long sessions, and map rendering all have room to get faster. Especially on older devices.
  • Polish. Small UI inconsistencies, accessibility improvements, and the kind of fit-and-finish work that's hard to prioritize during a feature push.

This doesn't mean zero new features forever — it means the bar for shipping something new goes up. If it's not a clear win for operators in the field, it can wait.

Thanks for using Carrier Wave. 73.
