iOS State of the Union

Two releases shipped in two weeks, a comprehensive manual rewrite, and an App Store launch that went live. Here's where Carrier Wave stands heading into April — and why you'll see us shift gears toward stabilization.

What shipped

March was one of the most productive months since development started. Two point releases landed back-to-back:

1.76.0 — Radio Control & Session Management

  • WiFi radio control. Carrier Wave can now connect to your rig over WiFi in addition to Bluetooth LE. WiFi is more reliable during transmit (no BLE dropouts) and supports higher data throughput for scope and waterfall data. Icom CI-V radios like the IC-705 work in both Access Point and Station mode.
  • Session templates. Save a session configuration — mode, frequency, equipment, activation type — as a named template and recall it with a tap. Separate from "Save as Defaults" so you can keep multiple setups for different operating scenarios.
  • SMS spotting. For off-grid POTA and SOTA activations where you have cell service but no data, you can now self-spot via text message.
  • Repeating auto-spot. SPOT <message> REPEAT <min> keeps your spot fresh on the network. SPOT STOP cancels it.
  • Session QSO comments. Set a comment that gets prepended to every QSO's notes — useful for contest exchanges or consistent activation notes.
  • Edit session callsign. Change the station callsign on an existing session and it propagates to all QSOs in that session.
  • Experiments section in Settings. Alpha and beta features now get maturity badges so you know what to expect.

1.77.0 — Spot Filtering

  • Persistent filters. HUNT and P2P filter selections now save across sessions. Active filters show filled icons and a descriptive summary (e.g., "20m, CW, POTA only").
  • Club and friends filtering. In HUNT mode, filter spots to show only members of your clubs or people on your friends list.
  • Activity filter. Narrow hunt spots by program — POTA, SOTA, or all.
  • P2P band/mode filters. Multi-select band and mode filters for park-to-park spots.
  • ADIF import into sessions. Long-press any session to import an ADIF file directly, merging contacts with validation warnings for callsign or park reference mismatches.

Beyond the app

The two weeks weren't just code. A few things happened around the product:

  • App Store launch. Carrier Wave went live at $4.99. We updated the website, removed "Coming Soon" messaging, and pointed everything at the real App Store listing.
  • Comprehensive manual rewrite. The documentation got a ground-up overhaul — 9 new reference pages (FT8, Radio Control, Activations, Contests, Smart Needs, Brag Sheets, Video Export, Events, Onboarding), and 17 existing pages updated. Three persona-specific paths guide new users, beginners, and power operators through different depths of the app.
  • Challenges removed. We pulled the Challenges feature entirely. It wasn't adding enough value for the complexity it introduced, and we'd rather focus on the features operators actually use every day.
  • Website cleanup. Removed open-source claims and GitHub links from the site to better reflect the current direction of the project.

Where we are

Carrier Wave is at v1.77.0 with a feature set that covers the full lifecycle of an operating session: rig control, logging, spotting, syncing, activations, contests, social features, and analytics. The app talks to QRZ, POTA, LoTW, HAMRS, eQSL, and HamAlert. It runs on iPhone, iPad, and Apple Watch.

That's a lot of surface area — and it's the right time to stop adding to it for a while.

What's next: stabilization

Starting in April, the focus shifts from new features to making everything that's already there rock solid. Concretely, that means:

  • Bug fixes over features. If it crashes, drops a connection, or loses data, that gets fixed first. New feature work takes a back seat.
  • Sync reliability. With seven services connected, sync edge cases are the biggest source of user-reported issues. We're going to methodically work through the backlog.
  • Performance. Large logs, long sessions, and map rendering all have room to get faster. Especially on older devices.
  • Polish. Small UI inconsistencies, accessibility improvements, and the kind of fit-and-finish work that's hard to prioritize during a feature push.

This doesn't mean zero new features forever — it means the bar for shipping something new goes up. If it's not a clear win for operators in the field, it can wait.

Thanks for using Carrier Wave. 73.
